Principe de calcul

Conditions environnement

Saisir les températures et humidités relatives des 3 chais. Les deltas sont calculés uniquement par rapport à la journée précédente.

Saisie quotidienne

Pour chaque chai : Médiane HR = HR saisie du jour ; ΔHR = HR du jour - HR de la veille ; Médiane T = T saisie du jour ; ΔT = T du jour - T de la veille. La première date sert de point de départ : Δ = 0.

Saisie des prévisions

Seuils d’alerte extérieure

Par défaut : alerte si T extérieure ≥ 30 °C ou HR extérieure ≤ 45 %. Tu peux ajuster ces seuils selon la sensibilité observée des chais.

Paramètres modèle

Modifier les coefficients de l'équation et les prix par cuvée.

Équation de perte de masse

Formule : perte g/barrique/j = constante + a×MédianeHR + b×DeltaHR + c×DeltaT + d×MédianeT.

Calcul des variables

MédianeHR = HR J saisie
DeltaHR = HR J - HR J-1
MédianeT = T J saisie
DeltaT = T J - T J-1

L’air extérieur n’est pas pris en compte. La première journée sert uniquement de référence : les deltas valent 0 pour cette première ligne.

Important : les classes de risque sont uniquement des indicateurs visuels. Elles ne modifient jamais la perte calculée. Le volume restant est calculé en ajoutant le cumul signé aux 225 L/barrique. L’impact économique utilise la valeur absolue du volume manquant.

Prix par bouteille

Conversion utilisée : 1 g ≈ 1 mL, 1 bouteille = 0,75 L. L’impact économique est calculé avec le prix de la cuvée.

Sauvegarde & partage

Sécuriser les données de l’application et éviter les versions différentes selon les ordinateurs.

État de la sauvegarde locale

Point important : les données de l’application sont enregistrées localement dans le navigateur. Deux ordinateurs peuvent donc avoir deux versions différentes. Pour travailler à plusieurs, il faut utiliser un fichier de sauvegarde JSON commun, stocké par exemple dans SharePoint/OneDrive, puis le restaurer avant de travailler.

Mode d’utilisation conseillé

SituationBonne pratique
Une seule personne travailleExporter une sauvegarde JSON à la fin de chaque modification importante.
Plusieurs personnes travaillentUne seule personne modifie à la fois. Elle restaure d’abord la dernière sauvegarde JSON, modifie, puis exporte une nouvelle sauvegarde.
Changement d’ordinateurRestaurer la dernière sauvegarde JSON avant d’utiliser l’application.
